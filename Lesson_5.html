Lesson 5 Dependency detection

Lesson 5 is the core lesson in this eLearning resource, it gives an technical description of strategies that tackle the complexity of dependency discovery tasks. The dependencies are divided into three categories: uniqueness, inclusion dependencies and functional dependencies. Other types of dependencies that are relevant to data profiling will also be discussed.

1. Unique column combinations and keys

Uniqueness refers to the unique value in unique column. It consists of four definition, in this definitions R and S denote relational schemata, with r and s denoting instances of R and S respectively.

1). Basic definition of uniqueness

  • Definition 1 (Unique) A column combination X ⊆ R is a unique, iff ∀ri , r j ∈ r, i = j : ri [X] = r j [X].
    Given a relation R with instance r, a unique column com- bination (a “unique”) is a set of columns X ⊆ R whose projection on r contains only unique value combinations.

  • Definition 2 (Non-unique) A column combination X ⊆ R is a non-unique, iff ∃ri , r j ∈ r, i = j : ri [X] = r j [X]
    Analogously, a set of columns X ⊆ R is a non-unique column combination (a “non-unique”), iff its projection on r contains at least one duplicate value combination.

  • Definition 3 (Minimal Unique) Acolumn combination X ⊆ R is a minimal unique, iff ∀X’ ⊂ X : X’ is a non-unique.

  • Definition 4 (Maximal Non-Unique) A column combination X ⊆ R is a maximal non-unique, iff ∀X’ ⊃ X : X’ is a unique.
    Acolumn combination X ⊆ R is a minimal unique, iff ∀X’ ⊂ X : X’ is a non-unique.
    Each superset of a unique is also unique while each subset of a non-unique is also a non-unique. Therefore, discovering all uniques and non-uniques can be reduced to the discovery of minimal uniques and maximal non-uniques.

2). Column-based traversal of the column lattice
  1. Apriori Algorithms
    In computer science and data mining, Apriori is a classic algorithm for learning association rules. Apriori is designed to operate on databases containing transactions. As is common in association rule mining, given a set of itemsets, the algorithm attempts to find subsets which are common to at least a minimum number C of the itemsets. Apriori uses a “bottom up” approach, where frequent subsets are extended one item at a time (a step known as candidate generation), and groups of candidates are tested against the data. The algorithm terminates when no further successful extensions are found.

2. Function dependencies

Fig. 4 Classification of Algorithms

1). Column-based algorithms
  • TANE Algorithm: TANE is based on partitioning the set of rows with respect to their attribute values, which makes testing the validity of functional dependencies fast even for a large number of tuples.1
  • FUN Algorithm: FUN, is a level-wise algorithm exploring, level after level, the attribute set lattice of the input relation. Fun handles, step by step, attribute sets of increased length and, at each new level, takes benefits from the knowledge acquired at the previous iteration.2
  • FD_Mine Algorithm: FD_Mine takes advantage of the rich theory of FDs to guide the search for FDs. More specifically, the use of FD theory can reduce both the size of the dataset and the number of FDs to be checked by pruning redundant data and skipping the search for FDs that follow logically from the FDs already discovered. 3
  • DFD Algorithm: DFD is uded to discover all functional dependencies in a dataset following a depth- first traversal strategy of the attribute lattice that combines aggressive pruning and efficient result verification.4
2). Row-based algorithms

  • Dep-Miner Algorithm: Dep-Miner a software prototype implementing a simple but effective model for the evaluation of itemsets, and in general for the evaluation of the dependencies between the variables on a domain of finite values. 5

  • FastFDs Algorithm: FastFDs Algorithm search strategy for generating minimal covers of hyp ergraphs. FastFDs Algorithm can be used to solve search problems in Arti cial Intelligence.6
